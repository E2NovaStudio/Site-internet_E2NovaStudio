Explications détaillées du code

Projet E2 Nova Studio - fichiers index.html, css/styles.css et js/main.js

Document généré pour comprendre la structure de la page d'accueil, son style visuel et les interactions JavaScript.

Remarque sur le nom du fichier CSS : la demande mentionne style.css, mais le fichier réellement utilisé par la page est css/styles.css, appelé à la ligne 11 de index.html.

Vue d'ensemble

index.html contient le contenu de la page : métadonnées, en-tête, navigation, sections visibles, cartes de services et pied de page.

css/styles.css définit l'identité visuelle : couleurs, grille, typographie, cartes, boutons, formulaire, animations visuelles de fond et responsive design.

js/main.js ajoute le comportement : menu mobile, soumission du formulaire de contact, recentrage des ancres et animation de particules dans le canvas.

index.html

Ce fichier est la page d'accueil. Il ne contient pas de logique complexe : il assemble les éléments sémantiques que le CSS habille et que le JavaScript rend interactifs.

css/styles.css

Le CSS agit comme un système visuel global. Plusieurs règles concernent aussi les autres pages du site, pas seulement index.html, par exemple les blocs de contact, les pages de services ou les mentions légales.

Fondations globales

js/main.js

Le JavaScript est écrit de manière défensive : chaque fonctionnalité vérifie d'abord que les éléments HTML existent. Cela permet au même fichier d'être chargé sur plusieurs pages sans provoquer d'erreurs.

Menu mobile

1-2 Le script récupère le bouton du menu mobile et la navigation principale avec document.querySelector.
4 La condition vérifie que les deux éléments existent avant d'ajouter des comportements.
5-8 closeNav ferme le menu en retirant la classe open et en remettant aria-expanded à false.
10-13 Au clic sur le bouton, classList.toggle("open") ouvre ou ferme le menu. La valeur retournée est convertie en texte pour l'attribut ARIA.
15-17 Chaque lien du menu ferme la navigation au clic. C'est pratique sur mobile après avoir choisi une page.
19-21 Si la fenêtre repasse au-dessus de 920 px, le menu mobile est fermé pour éviter un état incohérent au changement de taille.
23-25 La touche Échap ferme aussi le menu, ce qui améliore l'accessibilité clavier.

Formulaire de contact

28-32 Le script cherche le formulaire, le message de statut et le bouton d'envoi. Sur la page d'accueil, le formulaire n'existe pas, donc ce bloc ne s'exécute pas.
33-37 Lors de l'envoi, event.preventDefault() empêche le rechargement de page. Si le statut ou le bouton manque, la fonction s'arrête.
38-42 Le texte initial du bouton est conservé, le message est vidé, les classes de retour sont retirées, puis le bouton est désactivé avec le texte "Envoi en cours…".
43-49 Le bloc try prépare un FormData et envoie les données à Web3Forms via fetch en méthode POST.
50-54 Si la réponse HTTP échoue ou si Web3Forms renvoie success: false, une erreur est déclenchée avec un message explicite.
55-58 En cas de réussite, un message positif est affiché, la classe is-success est ajoutée et le formulaire est réinitialisé.
59-63 En cas d'erreur, le problème est écrit dans la console et un message d'échec est affiché à l'utilisateur avec la classe is-error.
64-67 Le bloc finally s'exécute dans tous les cas : il réactive le bouton et restaure son texte initial.

Ancres et défilement

71-72 centerHashTarget commence par vérifier si l'URL contient une ancre, par exemple #web-ia.
74 Le script cherche dans le document l'élément correspondant à cette ancre.
76-80 Si l'élément existe, un léger délai laisse la page se charger, puis scrollIntoView le place au centre de l'écran avec un défilement fluide.
83-84 La fonction est appelée au chargement, puis à chaque changement d'ancre dans l'URL.

Animation du canvas

86-94 Le script récupère le canvas, son contexte 2D, initialise la liste de particules, la position du pointeur, les dimensions et l'identifiant d'animation.
96-109 createParticles vide la liste puis crée un nombre de particules adapté à la taille de l'écran, entre 36 et 86.
101-107 Chaque particule reçoit une position aléatoire, une vitesse horizontale/verticale légère et une taille aléatoire.
111-121 resize adapte le canvas à la taille de la fenêtre et au ratio de pixels de l'écran. Le ratio est limité à 2 pour garder de bonnes performances.
123-130 drawLine dessine un segment bleu semi-transparent entre deux points. L'opacité est passée en paramètre.
132-147 render efface le canvas, déplace chaque particule, la fait réapparaître de l'autre côté si elle sort de l'écran, puis la dessine sous forme de cercle.
149-156 Chaque particule est comparée aux suivantes. Si deux particules sont proches de moins de 118 px, une ligne les relie avec une opacité proportionnelle à leur distance.
158-164 Si le pointeur est actif, les particules proches de la souris ou du doigt sont reliées au pointeur jusqu'à 150 px.
167 requestAnimationFrame relance render au prochain rafraîchissement de l'écran. C'est la boucle d'animation.
170-178 Les événements mettent à jour la taille du canvas, la position du pointeur et l'état actif/inactif du pointeur.
180-181 Le canvas est dimensionné une première fois, puis l'animation démarre.
183-189 Quand l'onglet devient masqué, l'animation est annulée pour économiser des ressources. Quand l'onglet redevient visible, elle reprend.

Relations entre les fichiers

  • index.html crée le canvas ligne 15, styles.css le place en arrière-plan lignes 77-85, et main.js l'anime lignes 86-190.
  • index.html crée le bouton mobile lignes 20-24, styles.css le dessine lignes 144-161 et l'affiche en mobile lignes 730-733, puis main.js le rend interactif lignes 1-26.
  • Les liens de services de index.html lignes 71, 80 et 89 pointent vers des ancres de services.html. main.js améliore leur positionnement à l'écran lignes 71-84.
  • Les classes comme button, service-card, cta-band et hero sont déclarées dans le HTML puis transformées visuellement par le CSS.

Points à surveiller

  • Les lignes 322-390 du CSS contiennent des styles de composants qui ne sont pas utilisés directement dans index.html, mais peuvent servir aux autres pages.
  • Les styles .form-status.is-success et .form-status.is-error sont actuellement placés dans la media query max-width: 430px. Sur desktop, les classes sont ajoutées par JavaScript, mais les couleurs spécifiques de succès/erreur ne s'appliquent pas forcément.
  • La ligne 116 du CSS est plus indentée que les autres propriétés du bloc .brand img. Ce n'est pas bloquant, mais un formatage homogène améliorerait la lisibilité.
